# Story 3.3: Favicon - Tematický Ikon pro Aplikaci

## Story ID
3.3

## Title
Favicon - Malý Ikon Hry v Browser Tabu

## Epic
3

## Status
Completed

## Description
Vytvořit a nainstalovat favikon (favicon.ico) pro aplikaci. Favikon bude tematicky vázaný na hru Piškvorky - něco s hrací deskou nebo X/O značkami.

Cíl: Když si uživatel otevře aplikaci, bude vidět malý custom ikon v browser tabu místo generického ikonu.

## Functional Requirements Covered
- FR1: Favikon aplikace je dostupný
- FR2: Favikon je tematický (související s Piškvorky)
- FR3: Favikon se zobrazuje v browser tabu

## Acceptance Criteria

- AC3.3: Favikon je dostupný na `/favicon.ico`
- AC3.3.1: Soubor existuje: `/src/favicon.ico` (nebo `/favicon.png`, `/favicon.svg`)
- AC3.3.2: Velikost: 32x32 pixelů (nebo víceúčelový, více velikostí)
- AC3.3.3: Format: .ico (nebo .png, .svg)
- AC3.4: HTML reference je správná
- AC3.4.1: V index.html je: `<link rel="icon" type="image/x-icon" href="/favicon.ico">`
- AC3.4.2: Cesta je absolutní (od rootu aplikace)
- AC3.5: Favikon se zobrazuje v browser tabu
- AC3.5.1: Po reload stránky vidíš ikon v tabu
- AC3.5.2: Ikon je vidět v bookmarku

- AC3.6: Favikon je součástí dist/ buildu

## Technical Notes

### Favikon Formáty

Možnosti (v pořadí preference):

1. **ICO (classic)**: `favicon.ico`
   - Tradiční formát
   - Obecná podpora
   - Ideální pro starší browsers

2. **PNG (moderní)**: `favicon.png`
   - Lepší kvalita
   - Menší velikost
   - Vyžaduje modernější browser

3. **SVG (best)**: `favicon.svg` ✓ VYBRANÉ
   - Bezkonečná škálovatelnost
   - Nejmenší velikost
   - Nejnovější formát
   - Perfektní pro hru s geometrickými tvary

### HTML Reference

```html
<!-- SVG favicon (vybrané) -->
<link rel="icon" type="image/svg+xml" href="/favicon.svg">
```

### Tematické Nápady

- **Hrací deska**: Malá mřížka 15x15 (nebo 3x3) ✓ IMPLEMENTOVÁNO
- **X a O**: Velké X a O v kontrastu
- **Kombinace**: Čárkovaná deska s X/O
- **Minimalistický**: Jednoduché symboly
- **Pixel art**: Retro nádech

### Vite Build

Vite automaticky kopíruje soubory z `public/` do `dist/`:
- Umisti favicon do `public/`
- Vite jej zkopíruje do `dist/assets/` při buildu
- Automaticky přidá hash ke jménu pro cache-busting
- Nemusíš nic měnit v `vite.config.js`
