# Story 4.11: Game Size Logic

## Epic
Epic 4: Design & UI Refresh

## User Story

Jako developer
Chci, aby logika hry byla adaptibilní na různé velikosti mřížky
Abych mohl podporovat 3×3, 10×10 i 15×15 hry

## Description

Změna herní logiky pro podporu dynamické velikosti:
- Konstruktor `TicTacToeGame(size)` s parametrem
- `winningLength` dle velikosti (3 pro 3×3, 5 pro jiné)
- `bombCount` dle velikosti (1, 10, 15)
- Dynamické renderování gridu

## Acceptance Criteria

1. **AC1:** TicTacToeGame konstruktor akceptuje `size` (3, 10, 15)
2. **AC2:** winningLength je 3 pro 3×3, 5 pro ostatní
3. **AC3:** bombCount je 1 pro 3×3, 10 pro 10×10, 15 pro 15×15
4. **AC4:** `renderGrid()` vytvoří správný počet buněk (9, 100, 225)
5. **AC5:** Všechny testy procházejí
