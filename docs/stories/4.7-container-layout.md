# Story 4.7: Container Layout Refactoring

## Epic
Epic 4: Design & UI Refresh

## User Story

Jako developer
Chci, aby `.game-container` měl rozumné max-width omezení
Abych měl lepší kontrolu nad layoutem a škálováním

## Description

Refaktoring CSS pro `.game-container`. Aktuálně má redundantní `width: 100%` a `max-width: 100%`, která oba znamenají "natáhni se na 100% rodičovské šířky".

Lepší přístup:
- Odebrat `width: 100%` a `max-width: 100%`
- Přidat rozumné `max-width` (650px - stejný jako mřížka)
- Přidat `margin: 0 auto` pro horizontální vycentrování

**Výsledek:** Container se nebude natahovat do nekonečna na velkých displejích.

## Technical Notes

```css
/* BEFORE */
.game-container {
    background: white;
    border-radius: 1rem;
    box-shadow: 0 20px 60px rgba(0, 0, 0, 0.3);
    padding: 1.5rem;
    max-width: 100%;  /* Redundantní! */
    width: 100%;      /* Redundantní! */
}

/* AFTER */
.game-container {
    background: white;
    border-radius: 1rem;
    box-shadow: 0 20px 60px rgba(0, 0, 0, 0.3);
    padding: 1.5rem;
    max-width: 650px;     /* Stejný jako .grid-container */
    margin: 0 auto;       /* Horizontální center */
    width: 100%;          /* Vyplnit do max-width */
}
```
